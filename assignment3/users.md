Garv was responsible for designing the visual structure and interface flow of the AI-Powered Placement Management Platform. His work focused on creating organized, user-friendly, and intuitive layouts for different stakeholders using wireframes and screen structure planning.

## Key Contributions

### Step 1: Requirement Understanding
- Studied the overall platform objectives and identified the important screens required for the system.
- Analyzed how students, recruiters, and administrators would interact with the platform.

### Step 2: Screen Identification
- Identified key interfaces including:
  - Login and Registration Pages
  - Student Dashboard
  - Recruiter Dashboard
  - Placement Tracking Screen
  - AI Recommendation Section
  - Admin Management Panel

### Step 3: Wireframe Design
- Designed low-fidelity wireframes representing:
  - Navigation bars
  - Menus
  - Buttons
  - Cards
  - Forms
  - Dashboard structures

### Step 4: User Experience Planning
- Organized layouts to ensure:
  - smooth navigation
  - proper information flow
  - user-friendly interaction
  - accessibility and usability

### Step 5: Design Review
- Refined interface layouts and contributed suggestions for improving platform usability and overall visual consistency.

# Gaurangi

## Role: Feature Prioritization & Analysis

Gaurangi was responsible for identifying, analyzing, and prioritizing the core functionalities of the AI-Powered Placement Management Platform. The work focused on understanding which features were essential for the platform’s MVP and long-term development.

## Key Contributions

### Step 1: Requirement Analysis
- Studied the project objectives and analyzed the major functionalities required for the platform.
- Identified the needs of students, recruiters, placement coordinators, and administrators.

### Step 2: Feature Identification
- Prepared a list of important platform features including:
  - AI-based Job Recommendations
  - Resume Upload & Analysis
  - Application Tracking
  - Recruiter Management
  - Notifications & Alerts
  - Interview Scheduling
  - Placement Analytics

### Step 3: Feature Categorization
- Classified features into:
  - High Priority
  - Medium Priority
  - Low Priority
- Categorized features based on:
  - system importance
  - user impact
  - usability
  - implementation necessity

### Step 4: MVP Planning
- Identified the features required for the Minimum Viable Product (MVP).
- Focused on selecting the essential functionalities necessary for the first version of the platform.

### Step 5: Documentation
- Documented the prioritized features in a structured format with descriptions and justifications for development planning.
